# ❓ Часто задаваемые вопросы (FAQ)

## Регистрация на выставку

### Q: Как visitor регистрируется на выставку?
**A:** Есть 2 способа:
1. На главной странице нажать "Зарегистрироваться" на карточке выставки
2. Перейти на страницу выставки `/exhibitions/[id]` и нажать кнопку в сайдбаре

После этого откроется трехэтапное модальное окно.

### Q: Что входит в форму регистрации?
**A:** На первом шаге нужно заполнить:
- Имя
- Фамилия
- Email
- Телефон

На втором шаге выбрать город проведения выставки.

### Q: Как работает генерация QR-кода?
**A:** QR-код генерируется автоматически через API qrserver.com.
QR содержит зашифрованные данные:
```json
{
  "registrationId": "REG-1704067200000",
  "name": "Иван Петров",
  "email": "ivan@example.com",
  "phone": "+7 (999) 123-45-67",
  "exhibition": "TechExpo 2024",
  "city": "Москва",
  "date": "2024-01-28T12:34:56Z"
}
```

### Q: Где visitor видит свои QR-коды?
**A:** В личном кабинете `/participant` → вкладка "Мои выставки (QR)"

Там отображаются:
- Все регистрации
- QR-коды
- Информация о регистрации
- Кнопки скачивания и показа на экране

### Q: Сколько времени занимает регистрация?
**A:** Примерно 2-3 минуты:
- Заполнение формы: 1 минута
- Выбор города и обработка: 1-1.5 минуты (показывается лоадер)
- Успешная регистрация: мгновенно

## Интеграция с Bitrix24

### Q: Когда отправляются данные в Bitrix?
**A:** Данные отправляются сразу после выбора города на втором шаге регистрации.

Отправляются:
- Фамилия
- Имя
- Email
- Номер телефона
- ID выставки
- Город

### Q: Как я узнаю, что данные отправились в Bitrix?
**A:** Откройте консоль браузера (F12 → Console) и посмотрите логи:
```
[v0] Отправка данных в Bitrix24:
{
  "firstName": "Иван",
  "lastName": "Петров",
  ...
}
```

### Q: Что делать, если Bitrix интеграция не работает?
**A:** Для реальной интеграции обновите файл `/lib/email-service.ts`:

Замените функцию `sendBitrixIntegration()`:
```typescript
export async function sendBitrixIntegration(data: {...}) {
  const response = await fetch('https://your-domain.bitrix24.ru/rest/1/YOUR_TOKEN/...', {
    method: 'POST',
    body: JSON.stringify({...})
  })
  return response.ok
}
```

## Email уведомления

### Q: Получит ли visitor письмо?
**A:** Да, письмо отправляется автоматически после успешной регистрации.

В разработке письмо логируется в консоль браузера. В production нужно настроить реальный email сервис.

### Q: Какие данные в письме?
**A:** Письмо содержит:
- Подтверждение регистрации
- Детали выставки
- ID регистрации
- QR-код (в реальном приложении)
- Информацию о проведении

### Q: Как настроить реальную отправку email?
**A:** Обновите файл `/lib/email-service.ts`:

Используйте SendGrid, Mailgun или другой сервис:
```typescript
import sgMail from '@sendgrid/mail'

export async function sendRegistrationEmail(data: EmailData) {
  const msg = {
    to: data.to,
    from: 'noreply@myfair.com',
    subject: data.subject,
    html: generateEmailTemplate(data),
  }
  await sgMail.send(msg)
  return true
}
```

## Админ-панель

### Q: Как войти в админ-панель?
**A:** Используйте аккаунт администратора:
```
Email:    admin@example.com
Password: password
```

После входа перейдите на `/admin`

### Q: Какие разделы есть в админ-панели?
**A:** 5 основных разделов:

1. **Пользователи** (`/admin/users`)
   - Модерация
   - Блокировка/разблокировка
   - Удаление пользователей

2. **Публикации** (`/admin/publications`)
   - Управление выставками
   - Управление новостями
   - Редактирование и публикация

3. **Создать пользователя** (`/admin/create-user`)
   - Добавление новых пользователей
   - Выбор роли

4. **Отчеты** (`/admin/reports`)
   - Статистика регистраций
   - Анализ выставок
   - Графики

5. **Логи** (`/admin/logs`)
   - История действий
   - Фильтрация
   - Поиск

### Q: Как отредактировать выставку?
**A:**
1. Перейти на `/admin/publications`
2. Найти выставку в списке
3. Нажать на три точки (меню)
4. Выбрать "Редактировать"
5. Заполнить форму и сохранить

### Q: Как создать новую выставку?
**A:**
1. Перейти на `/admin/publications`
2. Нажать кнопку "+ Новая выставка"
3. Заполнить форму:
   - Название
   - Описание
   - Место проведения
   - Дата начала и окончания
4. Сохранить
5. Выбрать "Опубликовать" из меню

## Безопасность

### Q: Безопасно ли отправлять личные данные через форму?
**A:** В текущей версии (development) все происходит локально в браузере.

Для production нужно:
- ✅ HTTPS соединение
- ✅ Валидация на сервере
- ✅ Шифрование в БД
- ✅ CSRF защита
- ✅ Rate limiting

### Q: Как защищены маршруты?
**A:** Использутся `<ProtectedRoute>` компоненты:
```tsx
<ProtectedRoute requiredRoles={['admin']}>
  <AdminPage />
</ProtectedRoute>
```

Неавторизованные пользователи перенаправляются на `/auth/login`.

## Данные и хранилище

### Q: Где хранятся данные?
**A:** В текущей версии данные хранятся в памяти приложения через `AdminContext`.

Это означает:
- ✅ Данные исчезают при перезагрузке страницы
- ✅ Нет сохранения на сервер
- ✅ Подходит для демонстрации

Для production нужна реальная БД (PostgreSQL, MongoDB и т.д.).

### Q: Как подключить реальную базу данных?
**A:** Замените `AdminContext` на Supabase, Neon или другую БД:

```tsx
// Вместо useState, используйте запросы к БД
const [registrations, setRegistrations] = useState([])

useEffect(() => {
  // Запросить регистрации с сервера
  const data = await fetchRegistrations()
  setRegistrations(data)
}, [])
```
