# Цветовая схема и компоненты Myfair

## Система цветов (OKLCH формат)

Приложение использует современный формат цветов **OKLCH** для обеспечения согласованности цветов в светлой и темной темах.

### Основные цвета (Light Mode)

```css
--background: oklch(1 0 0)           /* Белый фон */
--foreground: oklch(0.12 0 0)        /* Черный текст */
--card: oklch(1 0 0)                 /* Белые карточки */
--card-foreground: oklch(0.12 0 0)   /* Черный текст на карточках */
```

**Используется для:** Основного фона страниц, белых карточек, общего фона приложения.

---

### Цвет PRIMARY (Фиолетовый)

```
Light:  oklch(0.45 0.15 280)  → Насыщенный фиолетовый (#6B5FFF примерно)
Dark:   oklch(0.65 0.2 280)   → Светлый фиолетовый (#A78BFF примерно)
```

**Используется для:**
- Основные кнопки (default variant)
- Primary CTA (Call-To-Action) элементы
- Активные элементы навигации
- Акцентные элементы дизайна
- Primary links

---

### Цвет SECONDARY (Оранжевый/Рыжий)

```
Light:  oklch(0.55 0.18 15)   → Насыщенный оранжевый (#FF8C42 примерно)
Dark:   oklch(0.75 0.22 15)   → Светлый оранжевый (#FFB366 примерно)
```

**Используется для:**
- Secondary кнопки
- Статистика и метрики
- Вторичные акценты
- Выделение важных данных

---

### Цвет ACCENT (Фиолетовый светлый)

```
Light:  oklch(0.65 0.2 280)   → Светлый фиолетовый (#A78BFF примерно)
Dark:   oklch(0.75 0.25 280)  → Еще светлее (#C4A8FF примерно)
```

**Используется для:**
- Hover состояния ghost кнопок
- Фокусные состояния
- Подсветка на фоне
- Interactive элементы

---

### Цвет DESTRUCTIVE (Красный)

```
Light:  oklch(0.55 0.2 25)    → Красный (#FF4444 примерно)
Dark:   oklch(0.65 0.2 25)    → Светлый красный (#FF7777 примерно)
```

**Используется для:**
- Кнопки удаления
- Кнопки с деструктивными действиями
- Ошибки и предупреждения
- Блокировка пользователя
- Опасные операции

---

### Вспомогательные цвета

#### MUTED (Серый)
```
Light:  oklch(0.93 0 0)       → Светлый серый (#E9E9E9)
Dark:   oklch(0.3 0 0)        → Темный серый (#4D4D4D)
```
**Используется для:** Disabled состояния, неактивные элементы, placeholder текст

#### BORDER (Граница)
```
Light:  oklch(0.9 0 0)        → Очень светлый серый (#E0E0E0)
Dark:   oklch(0.25 0 0)       → Темный серый (#404040)
```
**Используется для:** Границы карточек, линии разделители, outline кнопки

#### INPUT (Поле ввода)
```
Light:  oklch(0.96 0 0)       → Почти белый (#F5F5F5)
Dark:   oklch(0.2 0 0)        → Очень темный (#333333)
```
**Используется для:** Фон input полей, textarea, формы

## Варианты кнопок

### 1. Default Button (Основная кнопка)

```
Стиль: bg-primary text-primary-foreground hover:bg-primary/90

Light Mode:
  Фон: Фиолетовый (#6B5FFF)
  Текст: Белый (#FFFFFF)
  Hover: Фиолетовый темнее (#5548FF)

Dark Mode:
  Фон: Фиолетовый светлый (#A78BFF)
  Текст: Черный (#000000)
  Hover: Фиолетовый еще светлее (#B89EFF)
```

**Используется для:**
- "Зарегистрироваться"
- "Сохранить"
- "Создать"
- Главные CTA кнопки

**Пример использования:**
```tsx
<Button>Зарегистрироваться</Button>
<Button variant="default">Сохранить</Button>
```

---

### 2. Destructive Button (Кнопка удаления)

```
Стиль: bg-destructive text-white hover:bg-destructive/90

Light Mode:
  Фон: Красный (#FF4444)
  Текст: Белый (#FFFFFF)
  Hover: Красный темнее (#E63030)

Dark Mode:
  Фон: Красный темнее (#FF5555)
  Текст: Белый (#FFFFFF)
  Hover: Красный светлее (#FF8888)
```

**Используется для:**
- "Удалить пользователя"
- "Удалить выставку"
- Опасные операции
- Отклонить/Отмена деструктивных действий

**Пример использования:**
```tsx
<Button variant="destructive">Удалить</Button>
<Button variant="destructive" size="sm">Удалить</Button>
```

---

### 3. Outline Button (Контурная кнопка)

```
Стиль: border bg-background shadow-xs hover:bg-accent hover:text-accent-foreground

Light Mode:
  Граница: Серая (#E0E0E0)
  Фон: Белый (#FFFFFF)
  Текст: Черный (#1F1F1F)
  Hover Фон: Светлый фиолетовый (#F0E8FF)
  Hover Текст: Фиолетовый (#A78BFF)

Dark Mode:
  Граница: Темная серая (#404040)
  Фон: Почти черный (#333333)
  Текст: Белый (#FFFFFF)
  Hover Фон: Фиолетовый темный (#4D3D8A)
  Hover Текст: Белый (#FFFFFF)
```

**Используется для:**
- "Подробнее"
- "Отмена"
- Вторичные действия
- Link-подобные кнопки
- "Скачать"

**Пример использования:**
```tsx
<Button variant="outline">Подробнее</Button>
<Button variant="outline">Отмена</Button>
```

---

### 4. Secondary Button (Вторичная кнопка)

Стиль: bg-secondary text-secondary-foreground hover:bg-secondary/80

Light Mode:
  Фон: Оранжевый (#FF8C42)

Dark Mode:
  Фон: Оранжевый светлый (#FFB366)
  Текст: Черный (#000000)
  Hover: Оранжевый еще светлее (#FFC480)
```

**Используется для:**

- "Добавить"

**Пример использования:**

```

---

### 5. Ghost Button (Прозрачная кнопка)

```
Стиль: hover:bg-accent hover:text-accent-foreground

Light Mode:
  Фон: Прозрачный
  Текст: Черный (#1F1F1F)
  Hover Фон: Светлый фиолетовый (#F0E8FF)
  Hover Текст: Фиолетовый (#A78BFF)

Dark Mode:
  Фон: Прозрачный
  Текст: Белый (#FFFFFF)
  Hover Фон: Фиолетовый темный (#4D3D8A)
  Hover Текст: Белый (#FFFFFF)
```

**Используется для:**
- Иконки меню (три точки)
- Кнопки в таблицах
- Минималистичные действия
- "Еще" кнопки

**Пример использования:**
```tsx

<Button variant="ghost" size="icon"><MoreHorizontal /></Button>
```

---

### 6. Link Button (Ссылка как кнопка)

```

Light Mode:
  Текст: Фиолетовый (#6B5FFF)
  Подчеркивание: На hover

  Текст: Фиолетовый светлый (#A78BFF)
  Подчеркивание: На hover
```

**Используется для:**
- Текстовые ссылки
- "Забыли пароль?"
- Дополнительные ссылки
- Минималистичные действия

**Пример использования:**
```tsx
<Button variant="link">Читать подробнее</Button>
```

## Состояния кнопок

Все кнопки имеют следующие состояния:

### Hover
Кнопка меняет цвет фона или становится светлее на 10%

### Focus
```
border-ring focus-visible:ring-ring/50 focus-visible:ring-[3px]
```
Красивое кольцо фокуса с первичным цветом

### Active/Disabled
```
disabled:pointer-events-none disabled:opacity-50
```
Кнопка становится полупрозрачной и не кликается

### Aria-invalid (Ошибка)
```
aria-invalid:ring-destructive/20
aria-invalid:border-destructive
```
Красное кольцо для некорректных данных

## Использование в компонентах

### Card (Карточка)
```css
--card: oklch(1 0 0)              /* Белая карточка */
--card-foreground: oklch(0.12 0 0) /* Черный текст */
```

### Badge (Значок)
Использует вариант default с:
- `bg-primary text-primary-foreground` для filled
- `border border-primary text-primary` для outline

### Input (Поле ввода)
```css
--input: oklch(0.96 0 0)    /* Светлый фон */
--ring: oklch(0.45 0.15 280) /* Фиолетовый при фокусе */
```

### Alert (Оповещение)
```css
Успех: bg-green-50 text-green-900
Ошибка: bg-destructive/10 text-destructive
Информация: bg-primary/10 text-primary
```

## Темная тема (Dark Mode)

Все цвета автоматически переключаются в блоке `.dark`:

```css
.dark {
  --background: oklch(0.1 0 0)      /* Черный вместо белого */
  --foreground: oklch(0.95 0 0)     /* Белый вместо черного */
  /* ... и так далее */
}
```

Чтобы применить темную тему, просто добавьте класс `.dark` на `<html>` элемент:

```tsx
<html className={isDarkMode ? 'dark' : ''}>
```

## Практические примеры

### Кнопка для основного действия
```tsx
<Button className="w-full">Зарегистрироваться на выставку</Button>
```
Использует PRIMARY фиолетовый, привлекает внимание.

### Группа действий
```tsx
<div className="flex gap-2">
  <Button>Сохранить</Button>
  <Button variant="outline">Отмена</Button>
</div>
```

### Кнопка в таблице
```tsx
<Button variant="ghost" size="sm">
  <MoreHorizontal className="w-4 h-4" />
</Button>
```

### Деструктивное действие
```tsx
<Button variant="destructive" onClick={handleDelete}>
  <Trash2 className="w-4 h-4 mr-2" />
  Удалить пользователя
</Button>
```

## Дополнительные дизайн токены

### Radiuses (Радиусы скругления)
```css
--radius: 0.625rem (10px)
--radius-sm: 6px
--radius-md: 8px
--radius-lg: 10px
--radius-xl: 14px
```

### Charts (Цвета для графиков)
```css
--chart-1: oklch(0.45 0.15 280) /* Фиолетовый */
--chart-2: oklch(0.55 0.18 15)  /* Оранжевый */
--chart-3: oklch(0.65 0.2 280)  /* Светлый фиолетовый */
--chart-4: oklch(0.5 0.12 60)   /* Желтоватый */
--chart-5: oklch(0.6 0.15 190)  /* Голубой */
```

Используются для отображения статистики и графиков с разными цветами для каждого набора данных.
